Direct Alignment (DPO) & Rejection Sampling

Align language models from preferences — no reward model, no RL optimizer required.

The Key Insight: Skip the Reward Model

Classic RL path (Sections 5–7)

Preferences → Train RM → PPO / GRPO → πθ

Two-stage, expensive, needs on-policy rollouts.

Direct Alignment path

Preferences → DPO loss → πθ

One stage — reward is implicit in the policy log-ratio.

Core claim: The optimal policy π* under the KL-constrained RLHF objective has a closed form. We can reparameterise the reward in terms of πθ / πref — bypassing an explicit reward model entirely.

DPO Derivation (1/3): The Optimal Policy

The whole method hinges on one fact: the KL-constrained objective has a closed-form optimum. Blue = policy · cyan = ref · orange = reward · purple = β / Z / KL.

\[\max_{\htmlClass{k-policy}{\pi}}\; \mathbb{E}_{y\sim\htmlClass{k-policy}{\pi}}\!\bigl[\htmlClass{k-reward}{r(x,y)}\bigr] -\htmlClass{k-kl}{\beta}\,D_\mathrm{KL}\!\bigl(\htmlClass{k-policy}{\pi}\,\|\,\htmlClass{k-ref}{\pi_\mathrm{ref}}\bigr)\]
The KL-constrained RLHF objective: earn reward, stay near πref.
\[=\;\min_{\htmlClass{k-policy}{\pi}}\; \mathbb{E}_{y\sim\htmlClass{k-policy}{\pi}}\!\left[\log\frac{\htmlClass{k-policy}{\pi(y|x)}}{\htmlClass{k-ref}{\pi_\mathrm{ref}(y|x)}} -\tfrac{1}{\htmlClass{k-kl}{\beta}}\,\htmlClass{k-reward}{r(x,y)}\right]\]
Write $D_\mathrm{KL}=\mathbb{E}\bigl[\log\tfrac{\pi}{\pi_\mathrm{ref}}\bigr]$, divide by $-\htmlClass{k-kl}{\beta}$, and flip max → min.
\[=\;\min_{\htmlClass{k-policy}{\pi}}\; \mathbb{E}_{y\sim\htmlClass{k-policy}{\pi}}\!\left[\log\frac{\htmlClass{k-policy}{\pi(y|x)}}{\htmlClass{k-ref}{\pi_\mathrm{ref}(y|x)}\,e^{\htmlClass{k-reward}{r(x,y)}/\htmlClass{k-kl}{\beta}}}\right]\]
First fold the reward into the log: $-\tfrac{1}{\htmlClass{k-kl}{\beta}}\htmlClass{k-reward}{r}=-\log e^{\htmlClass{k-reward}{r}/\htmlClass{k-kl}{\beta}}$, so the denominator becomes $\htmlClass{k-ref}{\pi_\mathrm{ref}}\,e^{\htmlClass{k-reward}{r}/\htmlClass{k-kl}{\beta}}$ — not yet a normalized distribution.
\[=\;\min_{\htmlClass{k-policy}{\pi}}\; \mathbb{E}_{y\sim\htmlClass{k-policy}{\pi}}\!\left[\log\frac{\htmlClass{k-policy}{\pi(y|x)}}{\htmlClass{k-kl}{Z(x)}^{-1}\,\htmlClass{k-ref}{\pi_\mathrm{ref}(y|x)}\,e^{\htmlClass{k-reward}{r(x,y)}/\htmlClass{k-kl}{\beta}}}\right]-\log \htmlClass{k-kl}{Z(x)}\]
Now multiply & divide the denominator by $\htmlClass{k-kl}{Z(x)}=\sum_y \htmlClass{k-ref}{\pi_\mathrm{ref}}\,e^{\htmlClass{k-reward}{r}/\htmlClass{k-kl}{\beta}}$ — the denominator is now a valid distribution $\htmlClass{k-policy}{\pi^*}$, so the bracket is exactly $D_\mathrm{KL}(\htmlClass{k-policy}{\pi}\,\|\,\htmlClass{k-policy}{\pi^*})$; the leftover $-\log \htmlClass{k-kl}{Z}$ is independent of π.
\[\htmlClass{k-policy}{\pi^*(y|x)}=\frac{1}{\htmlClass{k-kl}{Z(x)}}\,\htmlClass{k-ref}{\pi_\mathrm{ref}(y|x)}\,\exp\!\Bigl(\tfrac{1}{\htmlClass{k-kl}{\beta}}\htmlClass{k-reward}{r(x,y)}\Bigr)\]
$D_\mathrm{KL}\ge 0$, zero iff π equals that denominator — so it is the optimum. Re-weights πref by $e^{\htmlClass{k-reward}{r}/\htmlClass{k-kl}{\beta}}$: higher reward ⇒ exponentially more mass.

DPO Derivation (2/3): Reward in Terms of the Policy

Invert that formula — solve for the reward instead of the policy.

\[\htmlClass{k-policy}{\pi^*(y|x)}=\frac{1}{\htmlClass{k-kl}{Z(x)}}\,\htmlClass{k-ref}{\pi_\mathrm{ref}(y|x)}\,\exp\!\Bigl(\tfrac{1}{\htmlClass{k-kl}{\beta}}\htmlClass{k-reward}{r(x,y)}\Bigr)\]
The closed form from the previous slide.
\[\log\htmlClass{k-policy}{\pi^*}=\log\htmlClass{k-ref}{\pi_\mathrm{ref}}+\tfrac{1}{\htmlClass{k-kl}{\beta}}\htmlClass{k-reward}{r(x,y)}-\log \htmlClass{k-kl}{Z(x)}\]
Take logs — the product becomes a sum.
\[\htmlClass{k-reward}{r(x,y)}=\htmlClass{k-kl}{\beta}\log\frac{\htmlClass{k-policy}{\pi^*(y|x)}}{\htmlClass{k-ref}{\pi_\mathrm{ref}(y|x)}}+\htmlClass{k-kl}{\beta}\log \htmlClass{k-kl}{Z(x)}\]
Rearrange for r: the reward is now an implicit function of the policy. $\htmlClass{k-kl}{\beta\log Z(x)}$ is an intractable per-prompt constant — but it is about to cancel.

DPO Derivation (3/3): Into Bradley–Terry → the Loss

Bradley–Terry needs only the reward difference — so the intractable $\htmlClass{k-kl}{Z(x)}$ disappears.

\[P(\htmlClass{k-pref}{y_w}\succ \htmlClass{k-pref}{y_l}\mid x)=\sigma\!\bigl(\htmlClass{k-reward}{r(x,\htmlClass{k-pref}{y_w})}-\htmlClass{k-reward}{r(x,\htmlClass{k-pref}{y_l})}\bigr)\]
Bradley–Terry (from reward modeling): the preference probability depends only on the reward difference.
\[\htmlClass{k-reward}{r(x,\htmlClass{k-pref}{y_w})}-\htmlClass{k-reward}{r(x,\htmlClass{k-pref}{y_l})}=\htmlClass{k-kl}{\beta}\log\frac{\htmlClass{k-policy}{\pi(\htmlClass{k-pref}{y_w}|x)}}{\htmlClass{k-ref}{\pi_\mathrm{ref}(\htmlClass{k-pref}{y_w}|x)}}-\htmlClass{k-kl}{\beta}\log\frac{\htmlClass{k-policy}{\pi(\htmlClass{k-pref}{y_l}|x)}}{\htmlClass{k-ref}{\pi_\mathrm{ref}(\htmlClass{k-pref}{y_l}|x)}}\]
Substitute $\htmlClass{k-reward}{r}=\htmlClass{k-kl}{\beta}\log\tfrac{\pi}{\pi_\mathrm{ref}}+\htmlClass{k-kl}{\beta\log Z(x)}$. Same prompt ⇒ the $\htmlClass{k-kl}{\beta\log Z(x)}$ terms are identical and cancel.
\[\mathcal{L}_\mathrm{DPO}=-\,\mathbb{E}_{(x,\htmlClass{k-pref}{y_w},\htmlClass{k-pref}{y_l})\sim\mathcal{D}}\!\left[\log\sigma\!\left(\htmlClass{k-kl}{\beta}\log\frac{\htmlClass{k-policy}{\pi_\theta(\htmlClass{k-pref}{y_w}|x)}}{\htmlClass{k-ref}{\pi_\mathrm{ref}(\htmlClass{k-pref}{y_w}|x)}}-\htmlClass{k-kl}{\beta}\log\frac{\htmlClass{k-policy}{\pi_\theta(\htmlClass{k-pref}{y_l}|x)}}{\htmlClass{k-ref}{\pi_\mathrm{ref}(\htmlClass{k-pref}{y_l}|x)}}\right)\right]\]
Maximize the likelihood of observed preferences → minimize this NLL. Replace π* with the trainable πθ. No reward model, no Z(x).

DPO Gradient: Push Up Chosen, Push Down Rejected

\[\nabla_\theta\mathcal{L}_\mathrm{DPO} = -\htmlClass{k-kl}{\beta}\,\mathbb{E}\!\left[ \underbrace{\sigma\!\left(\htmlClass{k-reward}{\hat r_\theta(x,y_l)}-\htmlClass{k-reward}{\hat r_\theta(x,y_w)}\right)}_{\text{mismatch weight }w} \!\cdot\! \bigl[\nabla_\theta\log\htmlClass{k-policy}{\pi_\theta(y_w|x)} - \nabla_\theta\log\htmlClass{k-policy}{\pi_\theta(y_l|x)}\bigr] \right] \]
  • Inner bracket: increase log-prob of yw, decrease log-prob of yl.
  • Weight w is large when the implicit reward is wrong — harder mistakes get bigger updates.
  • Implicit reward: $\htmlClass{k-reward}{\hat r_\theta(x,y)} = \htmlClass{k-kl}{\beta}\log\dfrac{\htmlClass{k-policy}{\pi_\theta(y|x)}}{\htmlClass{k-ref}{\pi_\mathrm{ref}(y|x)}}$

DPO vs. RL: Offline vs. Online Data

Fig (c) — Offline DPO (top) vs. online RL loop (bottom). RL generates fresh data each batch; DPO trains on a fixed dataset.

DPO (offline)

  • Fixed 𝒟 — no rollout loop
  • KL static: set exactly by β
  • Easier to implement; 2× less GPU with log-prob caching
  • Risk: training signal from other models → distribution mismatch

RL / PPO / GRPO (online)

  • Fresh on-policy samples each batch
  • KL tracked dynamically via penalty / clipping
  • Higher ceiling — especially for reasoning
  • More infrastructure: RM server, rollout workers
Verdict: RL methods edge ahead on benchmarks, but DPO dominates production (Llama 3, Tülu 3, Nemotron …) — data quality dominates algorithmic choice.

Rejection Sampling & Best-of-N

Generate N completions per prompt → score with a reward model → keep the best → SFT.

Top per prompt  $\hat{y}_i = \arg\max_j\; \htmlClass{k-reward}{r_{i,j}}$ — one best completion per prompt; balanced coverage.
Top-K overall — take the K highest-reward pairs across all prompts; maximises quality but may skew toward easy prompts.

Recap: Three Ways to Use Preferences

Reinforcement Learning

Explicit reward model → on-policy rollouts → PPO / GRPO.

Highest ceiling. Best for reasoning. Needs RM server + rollout workers.

DPO (Direct Alignment)

Implicit reward $\htmlClass{k-kl}{\beta}\log\frac{\htmlClass{k-policy}{\pi_\theta}}{\htmlClass{k-ref}{\pi_\mathrm{ref}}}$; offline on fixed preference pairs.

Simple, no RM. Widely used: Llama 3 Instruct, Tülu 3, Zephyr-β.

Rejection Sampling

Generate N → score with RM → SFT on top K. Best-of-N skips fine-tuning (inference-only).

Fewest moving parts. Used in WebGPT, Llama 2 Chat, OpenAI PRM paper.

All three minimise the same KL-constrained RLHF objective: \[\max_\pi\;\mathbb{E}\!\left[\htmlClass{k-reward}{r(x,y)}\right] - \htmlClass{k-kl}{\beta\,D_\mathrm{KL}(\pi\|\pi_\mathrm{ref})}\] They differ in how the reward is estimated and whether data is fixed or online.
Variants in the wild — IPO (softer target), SimPO (length-norm), ORPO (no reference model), online DPO (sample from the current policy). One caveat worth remembering: preference displacement — DPO can lower both log-probs, the rejected one more. Details → self-study.
