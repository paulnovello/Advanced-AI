The Frontier: Reasoning & RLVR When correctness replaces preferences

Standard RLHF needs a learned reward model to judge subjective quality. RLVR makes the reward model optional — a verification function returns a definitive 0/1 score.

Math

Model generates chain-of-thought. Answer extracted via \boxed{} or "The answer is:". Reward = 1 iff extracted answer matches ground truth.

Code

Model writes a function. Unit tests execute it. Reward = 1 if all assertions pass; 0 otherwise (or partial credit per test passed).

\[ \htmlClass{k-reward}{r}(y,\,y^*) \;\in\; \{0,\,1\} \]
Binary reward: no continuous score, no learned preference model.
\[ \htmlClass{k-reward}{r}(y,\,y^*) \;=\; \mathbb{1}\!\left[\,\text{verify}(y)\;=\;y^*\,\right] \]
Verify function checks the extracted answer against ground truth $y^*$ — a regex, unit test, or checker.
\[ \htmlClass{k-reward}{r}(y,\,y^*) \;=\; \mathbb{1}\!\left[\,\text{verify}(y)\;=\;y^*\,\right] \quad \longrightarrow \quad \text{plug into }\htmlClass{k-adv}{A_i}\text{ for GRPO} \]
Same advantage used in GRPO — reward source swapped, algorithm unchanged.

The RLVR Training Loop

Sample → verify → update. Repeat hundreds of epochs over the same verifiable problems.

\[ \text{RLHF:}\quad \htmlClass{k-adv}{A_i} \;\propto\; \htmlClass{k-reward}{r_{\phi}(x,y_i)} \;-\; \text{baseline} \]
Learned RM $r_\phi$ scores subjective quality.
\[ \text{RLVR:}\quad \htmlClass{k-adv}{A_i} \;\propto\; \htmlClass{k-reward}{\mathbb{1}[\text{verify}(y_i)\!=\!y^*]} \;-\; \text{baseline} \]
Verifier replaces RM. Same advantage formula, reward source swapped.
The unifying takeaway: GRPO + verifier reward = RLVR. Swap the reward source; keep the algorithm. RLHF and RLVR are one story. (KL is often removed in long RLVR runs — verifiable domains resist over-optimization.)

Why Now — and the New Scaling Axis

OpenAI o1 · DeepSeek R1

First public models trained with large-scale RLVR (2025) — the "thinking model" era. R1 (open weights, GRPO) reprioritized the entire industry.

Why it works now

Capability threshold: RLVR only lifts models that are already strong (~2024 base models cleared it). Tooling matured: TRL, veRL, OpenRLHF tamed RL's brittleness.

Two ways to spend test-time compute: deeper (longer reasoning chains) and wider (N parallel rollouts + majority vote). This is what RLVR training unlocks; regular RLHF models do not scale this way.

RLVR — Recap

  • RLVR = GRPO with a verifier instead of a learned reward model — 0/1 signal for math & code.
  • Loop: policy generates CoT → verifier scores → GRPO update → repeat (hundreds of epochs).
  • Works now because base models crossed a capability threshold (~2024) and tooling matured.
  • Two scaling axes: RL training (more gradient steps) + inference scaling (longer traces, more rollouts).
  • Same machinery as standard RLHF — only the reward source differs. RLHF and RLVR are one story.
Expanding beyond math/code — precise instruction following, multimodal, agentic tool use with execution feedback. Open questions (difficulty filtering, curricula, KL-free long runs) → self-study.
