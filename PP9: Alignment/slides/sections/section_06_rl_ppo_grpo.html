RL Optimization: Policy Gradient → PPO → GRPO

The RL Loop in RLHF

Two models are essential: a policy πθ (the LLM being trained) and a reward model that scores its outputs. A frozen copy serves as the reference model πref for the KL penalty.

Fig. (a) — The RLHF training loop: policy samples completions, reward model scores them, gradient updates close the loop.

Deriving the Policy Gradient

\[ \htmlClass{k-adv}{J(\theta)} = \mathbb{E}_{\tau\sim\htmlClass{k-policy}{\pi_\theta}}\!\bigl[\htmlClass{k-reward}{R(\tau)}\bigr] = \int_\tau \htmlClass{k-policy}{p_\theta(\tau)}\,\htmlClass{k-reward}{R(\tau)}\,d\tau \]
Objective: expected total reward over trajectories τ sampled from the current policy.
\[ \nabla_\theta \htmlClass{k-adv}{J(\theta)} = \int_\tau \nabla_\theta\htmlClass{k-policy}{p_\theta(\tau)}\,\htmlClass{k-reward}{R(\tau)}\,d\tau \]
Take the gradient; push ∇ inside the integral (standard for differentiable p_θ).
\[ \nabla_\theta\htmlClass{k-logp}{\log \htmlClass{k-policy}{p_\theta}} = \frac{\nabla_\theta \htmlClass{k-policy}{p_\theta}}{\htmlClass{k-policy}{p_\theta}} \;\;\Longrightarrow\;\; \nabla_\theta \htmlClass{k-policy}{p_\theta} = \htmlClass{k-policy}{p_\theta}\,\nabla_\theta\htmlClass{k-logp}{\log p_\theta} \]
The log-derivative identity — just the chain rule $\frac{d}{d\theta}\log f = f'/f$, multiplied through by $\htmlClass{k-policy}{p_\theta}$. That one rearrangement is the whole trick.
\[ \nabla_\theta \htmlClass{k-adv}{J} = \int_\tau \htmlClass{k-policy}{p_\theta}\,\nabla_\theta\htmlClass{k-logp}{\log p_\theta}\,\htmlClass{k-reward}{R}\,d\tau = \mathbb{E}_{\tau\sim\htmlClass{k-policy}{\pi_\theta}}\!\bigl[\nabla_\theta\htmlClass{k-logp}{\log p_\theta(\tau)}\,\htmlClass{k-reward}{R(\tau)}\bigr] \]
Substitute back: the factor $\htmlClass{k-policy}{p_\theta}$ turns $\int(\cdot)\,d\tau$ into an expectation under $\htmlClass{k-policy}{\pi_\theta}$ — now estimable by Monte-Carlo rollouts.
\[ \nabla_\theta \htmlClass{k-adv}{J(\theta)} = \mathbb{E}_{\tau\sim\htmlClass{k-policy}{\pi_\theta}}\!\left[\sum_{t}\nabla_\theta\htmlClass{k-logp}{\log \htmlClass{k-policy}{\pi_\theta(a_t|s_t)}}\;\htmlClass{k-adv}{\Psi_t}\right] \]
$\log p_\theta(\tau)$ expands into per-token policy terms (env. dynamics don't depend on θ). Replace $\htmlClass{k-reward}{R(\tau)}$ with a credit signal $\htmlClass{k-adv}{\Psi_t}$ — the policy gradient theorem.

REINFORCE: From Theorem to Algorithm

The theorem gives the gradient; REINFORCE is the simplest way to use it — pick Ψt, estimate by rollouts, step.

\[ \nabla_\theta J(\theta) = \mathbb{E}_{\tau\sim\htmlClass{k-policy}{\pi_\theta}}\!\left[\sum_{t} \nabla_\theta\log \htmlClass{k-policy}{\pi_\theta(a_t|s_t)}\;\htmlClass{k-adv}{\Psi_t}\right] \]
Start from the policy gradient theorem — Ψt is still an unspecified credit signal.
\[ \htmlClass{k-adv}{\Psi_t} \;=\; \htmlClass{k-adv}{G_t} \;=\; \sum_{t'=t}^{T}\gamma^{\,t'-t}\,\htmlClass{k-reward}{r_{t'}} \]
REINFORCE's one choice: use the actual sampled return-to-go Gt as the credit signal — no learned value, no model.
\[ \widehat{\nabla_\theta J} \;=\; \frac{1}{N}\sum_{i=1}^{N}\sum_{t} \nabla_\theta\log \htmlClass{k-policy}{\pi_\theta(a_t^{i}|s_t^{i})}\;\htmlClass{k-adv}{G_t^{i}} \]
Replace the expectation by a Monte-Carlo average over N sampled trajectories — this is computable.
\[ \theta \;\leftarrow\; \theta \;+\; \alpha\,\widehat{\nabla_\theta J} \]
Gradient ascent: nudge θ so high-return actions become more probable. Repeat.

REINFORCE in Practice

The same gradient, put to work in the LLM setting — and the catch (high variance) that motivates everything after.

The REINFORCE loop

  1. Roll out N trajectories with the current πθ
  2. Compute the return-to-go Gt at every step
  3. Estimate the gradient = mean of ∇log πθ · Gt
  4. Update θ ← θ + α ∇̂, then repeat

In the LLM setting

A trajectory is one generation; reward is terminal, so Gt = R(x,y) for every token: \[ \widehat{\nabla_\theta J} = \frac{1}{N}\sum_i \htmlClass{k-reward}{R(x,y^{i})}\sum_t \nabla_\theta\log\htmlClass{k-policy}{\pi_\theta(y_t^{i}\mid x,y_{<t}^{i})} \] Scale the log-likelihood of the whole sample by its reward.
Unbiased but high-variance: Gt is one noisy sample of the future. The fix — subtract a baseline b(s) — turns the return into an advantage and motivates everything that follows.

First, Two Value Functions: V and Q

A 30-second refresher — the vocabulary that turns REINFORCE's noisy return into a low-variance signal.

State value V(s)

\[ \htmlClass{k-value}{V^\pi(s)} = \mathbb{E}_{\htmlClass{k-policy}{\pi}}\!\bigl[\htmlClass{k-adv}{G_t}\mid s_t=s\bigr] \]

Expected return from state s if we follow π onward. How good is this situation?

Action value Q(s,a)

\[ \htmlClass{k-reward}{Q^\pi(s,a)} = \mathbb{E}_{\htmlClass{k-policy}{\pi}}\!\bigl[\htmlClass{k-adv}{G_t}\mid s_t=s,\,a_t=a\bigr] \]

Expected return if we take action a now, then follow π. How good is this action here?

\[ \htmlClass{k-value}{V(s)} = \mathbb{E}_{a\sim\htmlClass{k-policy}{\pi}}\!\bigl[\htmlClass{k-reward}{Q(s,a)}\bigr], \qquad \htmlClass{k-adv}{A(s,a)} = \htmlClass{k-reward}{Q(s,a)} \htmlClass{k-value}{- V(s)} \]
Why not just use REINFORCE's return Gt? Because Gt is one noisy sample of how the episode happened to unfold. Q(s,a) is its expectation — averaged over every way it could unfold. Swapping the sampled return for (an estimate of) Q keeps the same expected gradient but cuts the variance; subtracting V(s) gives the advantage A = Q − V — the next slide.

From Return to Advantage

Subtracting a state-dependent baseline reduces variance without changing the expected gradient. The optimal baseline is the advantage.

\[ \mathbb{E}_{a\sim\htmlClass{k-policy}{\pi(\cdot|s)}}\!\Bigl[b(s)\,\nabla_\theta\log \htmlClass{k-policy}{\pi_\theta(a|s)}\Bigr] = 0 \]
Any state-dependent baseline b(s) has zero expected gradient — subtracting it is free (doesn't bias the estimator), but it kills variance.
\[ \htmlClass{k-adv}{A(s_t, a_t)} = \htmlClass{k-reward}{Q(s_t, a_t)} \htmlClass{k-value}{- V(s_t)} \]
The optimal baseline is the value V(s): what remains is the advantage — how much better was this action than average?
\[ \htmlClass{k-adv}{A(s_t, a_t)} = \htmlClass{k-reward}{r_t} + \gamma\htmlClass{k-value}{V(s_{t+1})} \htmlClass{k-value}{- V(s_t)} \]
Bellman: Q = immediate reward + discounted future value. The TD residual — plug Ψ_t = A_t into the policy gradient. (γ = 1 in RLHF.)

Four completions for one prompt. Subtracting the baseline makes the signal relative: above-average up, below-average down. PPO learns this baseline with a critic $\htmlClass{k-value}{V(s_t)}$; GRPO will simply use the group mean.

Proximal Policy Optimization (PPO)

Taking multiple gradient steps on one data batch requires importance sampling. PPO clips the ratio to prevent runaway updates.

\[ \nabla_\theta J \!=\! \mathbb{E}_{\tau\sim\htmlClass{k-policy}{\pi_\theta}}\!\left[\sum_t\nabla_\theta\log \htmlClass{k-policy}{\pi_\theta(a_t|s_t)}\,\htmlClass{k-adv}{A_t}\right] \]
Advantage PG from the previous slide. The expectation is under the current πθ, so an honest estimate needs fresh rollouts every step. But generating rollouts is the expensive part — we'd rather take several update steps on one batch.
\[ J^{\text{IS}}(\theta) = \mathbb{E}_{\tau\sim\htmlClass{k-ref}{\pi_{\theta_{\text{old}}}}} \!\left[\frac{\htmlClass{k-policy}{\pi_\theta(a_t|s_t)}}{\htmlClass{k-ref}{\pi_{\theta_{\text{old}}}(a_t|s_t)}}\,\htmlClass{k-adv}{A_t}\right] \]
Reusing that batch makes it off-policy: after the first step πθ ≠ πθ_old, the policy that generated the data. Importance sampling corrects the mismatch — reweight each action by the ratio πθ/πθ_old so old samples still estimate the new expectation. That is where the ratio comes from.
\[ \htmlClass{k-policy}{\rho_t(\theta)} = \frac{\htmlClass{k-policy}{\pi_\theta(a_t|s_t)}}{\htmlClass{k-ref}{\pi_{\theta_{\text{old}}}(a_t|s_t)}} \;\longrightarrow\; \text{unbounded!} \]
The policy ratio ρ_t starts at 1 (πθ=πθ_old) — but it's unbounded: large ρ_t means destructively large updates.

Fig. (b) — PPO clip region. Blue (A>0): gradient zeroes when ρ>1+ε. Red (A<0): gradient zeroes when ρ<1−ε. Dot = moving ratio.

\[ J^{\text{PPO}}(\theta) = \mathbb{E}_t\!\left[\min\!\left(\htmlClass{k-policy}{\rho_t}\htmlClass{k-adv}{A_t},\; \operatorname{clip}\!\bigl(\htmlClass{k-policy}{\rho_t},\,\htmlClass{k-kl}{1{-}\varepsilon},\,\htmlClass{k-kl}{1{+}\varepsilon}\bigr)\htmlClass{k-adv}{A_t}\right)\right] \]
Clip ρ_t to [1±ε]; the min picks the smaller update — gradient is zero outside the trust region. In full PPO, A_t comes from a learned critic V(s_t) + GAE — a 4th model in memory. GRPO (next) removes it; GAE → self-study.

Group Relative Policy Optimization (GRPO)

Same clipped surrogate as PPO — but no value network. Instead, sample a group of G completions per prompt and compute advantages within the group.

\[ \text{PPO needs } \htmlClass{k-value}{V_\theta(s_t)}: \text{ a learned critic.} \]
PPO's baseline requires training a separate value network — extra memory, extra complexity.
\[ \{a_1, \ldots, a_G\} \sim \htmlClass{k-ref}{\pi_{\theta_{\text{old}}}}(\cdot\mid s),\;\; \htmlClass{k-reward}{r_i = \text{RM}(s, a_i)} \]
Generate G completions from the same prompt s; score each with the reward model. No gradient through the RM.
\[ \htmlClass{k-adv}{A_i} = \frac{\htmlClass{k-reward}{r_i} - \bar{r}}{\sigma_r} \]
Group-relative advantage: the group's mean r̄ and std σ_r replace the critic. How far above/below its group is completion i?

Fig. (c) — GRPO group sampling. One prompt fans out to G completions; advantages normalized within the group (no critic needed).

\[ J^{\text{GRPO}}(\theta) = \frac{1}{G}\sum_{i=1}^{G}\!\left(\min\!\left(\htmlClass{k-policy}{\rho_i}\htmlClass{k-adv}{A_i},\operatorname{clip}(\htmlClass{k-policy}{\rho_i},\htmlClass{k-kl}{1{\pm}\varepsilon})\htmlClass{k-adv}{A_i}\right) \htmlClass{k-kl}{-\beta D_{\mathrm{KL}}(\pi_\theta\|\pi_{\text{ref}})}\right) \]
Substitute A_i into the PPO-style objective. KL is added in the loss (not folded into the reward as in PPO).

PPO vs. GRPO — Side by Side

PPO

  • Advantage: learned value network Vθ per token (GAE, λ=0.95)
  • Models in memory: policy + πθ_old + value net + πref (4 copies)
  • KL placement: subtracted from per-token reward before advantage: $\tilde{r}_t = r_t - \beta\log\frac{\pi_\theta}{\pi_{\text{ref}}}$
  • Credit assignment: token-level — different At per token position
  • Primary use: InstructGPT, Tülu 3

GRPO

  • Advantage: group-relative — (ri−mean)/std over G completions per prompt
  • Models in memory: policy + πref only — no value network (~30–50% less memory)
  • KL placement: explicit loss term: $\mathcal{L} = \mathcal{L}_{\text{PG}} + \beta D_{\mathrm{KL}}(\pi_\theta\|\pi_{\text{ref}})$
  • Credit assignment: sequence-level — same Ai broadcast to all tokens in completion i
  • Primary use: DeepSeekMath, DeepSeek-R1, RLVR reasoning

Within the trust region (ρ ∈ [1±ε]) both algorithms reduce to the same policy-gradient update. They differ only in how they estimate A, where they put the KL, and how much memory they use.

A wider family exists — RLOO (leave-one-out baseline), GSPO (sequence-level ratios), CISPO (clip weights, not the objective), Dr.GRPO (no std-norm ≈ RLOO) — all share the gradient core $\Delta\theta \propto \Psi_t\nabla_\theta\log\pi_\theta$. Details → self-study.

KL Regularization & Recap

Every RLHF algorithm keeps πθ close to πref via a KL penalty — preventing reward hacking and preserving language quality. The two canonical placements:

PPO style — KL in reward

\[ \tilde{r}_t = \htmlClass{k-reward}{r_t} \htmlClass{k-kl}{- \beta \log\frac{\htmlClass{k-policy}{\pi_\theta(a_t|s_t)}}{\htmlClass{k-ref}{\pi_{\text{ref}}(a_t|s_t)}}} \]

KL subtracted from the per-token reward before advantage estimation → token-level credit assignment.

GRPO style — KL in loss

\[ \mathcal{L} = \mathcal{L}_{\text{PG}} \htmlClass{k-kl}{+ \beta\,D_{\mathrm{KL}}\!\left(\htmlClass{k-policy}{\pi_\theta} \;\|\; \htmlClass{k-ref}{\pi_{\text{ref}}}\right)} \]

KL added as a separate loss term → sequence-level advantage stays reward-only; token-level KL subtracted separately.

Section recap — derivation chain:
∇J = E[∇logπ·Ψ] → Ψ = A = Q−V = TD residual → PPO: clip(ρ, 1±ε)·A → GRPO: A=(r−mean)/std → KL keeps model sane

All algorithms reduce to $\Delta\theta \propto \Psi_t\,\nabla_\theta\log\pi_\theta(a_t|s_t)$. Differences are entirely in how $\Psi_t$ is estimated and how large a step is allowed.
