The LLM as an RL Problem

Standard RL was designed for control policies — how do we use it for language models?

Reinforcement Learning in 60 Seconds

A Markov Decision Process formalises sequential decision-making:

$\mathcal{M} = \langle \htmlClass{k-policy}{S},\; \htmlClass{k-policy}{A},\; \htmlClass{k-logp}{P},\; \htmlClass{k-reward}{r},\; \htmlClass{k-kl}{\gamma} \rangle$
  • $S$ — states; $A$ — actions; $P(s_{t+1}|s_t,a_t)$ — transitions
  • $r(s_t,a_t)$ — scalar reward; $\gamma\in[0,1)$ — discount. The return $\htmlClass{k-adv}{G_t}=\sum_{k\ge 0}\gamma^{k}\,r(s_{t+k},a_{t+k})$ is the discounted sum of future rewards.
  • agent picks actions with a policy $\pi(a|s)$; a rollout is a trajectory τ
\[\max_{\htmlClass{k-policy}{\pi}}\;\htmlClass{k-adv}{J(\pi)} = \max_{\htmlClass{k-policy}{\pi}}\;\mathbb{E}_{\tau\sim \htmlClass{k-policy}{p_\pi}}\! \Bigl[\textstyle\sum_{t} \htmlClass{k-kl}{\gamma^{t}}\,\htmlClass{k-reward}{r(s_t,a_t)}\Bigr]\]

The whole of RL in one line: find the policy that maximises expected discounted return.

Token Generation as a Trajectory

Each decoding step is one state → action → next state transition. Reward arrives only at the end — sparse & terminal.

State st = prompt + all tokens so far
Transition is deterministic: $s_{t+1} = [s_t \;||\; a_t]$
Action at = next token from vocabulary $\mathcal{V}$
πθ assigns a probability to each of ~50K tokens

Sparse Terminal Reward vs. Dense Reward

In classic RL, reward guides every step. In RLHF, the reward model scores the full completion — reward arrives only at the end.

Simplified RLHF objective (no horizon, no γ): $\displaystyle\max_{\pi}\;\mathbb{E}_{\tau\sim\pi}\!\bigl[\htmlClass{k-reward}{r_\theta(x,y)}\bigr]$

Three Ways the LLM MDP is Unusual

Massive action space

Vocabulary $|\mathcal{V}|\approx 32\text{K}$–$200\text{K}$ tokens. Tabular methods fail. We must generalize with neural networks — and unlike classic RL's random init, we fine-tune a pretrained LLM: a strong prior, not a tabula rasa.

Deterministic transitions

$s_{t+1} = [s_t \;||\; a_t]$ — fully determined by the token chosen. No environment stochasticity. Only the policy is random. Credit assignment is simpler (bandit, γ=1), but reward is entirely proxy-based.

Learned, hackable reward

$\htmlClass{k-reward}{r_\theta(x,y)}$ is itself a neural net, not a ground-truth environment signal. More flexible — but a strong optimizer will exploit its weaknesses. This demands a guard rail.

No state transitions. The prompt $x$ is sampled from a fixed dataset; the completion does not define the next prompt. One prompt + one completion = one complete episode.
A learned, hackable reward model demands a guard rail — the KL penalty. That is the topic of the next two slides.

Building the RLHF Objective

A step-by-step derivation — each color tracks the same concept across every line.

\[\max_{\htmlClass{k-policy}{\pi_\theta}}\; \mathbb{E}_{\htmlClass{k-policy}{\pi_\theta}}\!\bigl[ \htmlClass{k-reward}{r_\theta(x,y)} \bigr]\]
Start here. Maximize expected reward over completions $y \sim \htmlClass{k-policy}{\pi_\theta}(\cdot|x)$. Policy = blue; reward = orange.
\[\max_{\htmlClass{k-policy}{\pi_\theta}}\; \mathbb{E}_{\htmlClass{k-policy}{\pi_\theta}}\!\bigl[ \htmlClass{k-reward}{r_\theta(x,y)} \bigr] \htmlClass{k-kl}{- \;\beta \cdot (\,\cdots\,)}\]
Problem: unconstrained maximization exploits the proxy reward. Add a penalty term (purple) to control drift. $\htmlClass{k-kl}{\beta} > 0$ sets its strength.
\[\max_{\htmlClass{k-policy}{\pi_\theta}}\; \mathbb{E}_{\htmlClass{k-policy}{\pi_\theta}}\!\bigl[ \htmlClass{k-reward}{r_\theta(x,y)} \bigr] \htmlClass{k-kl}{-\;\beta\, D_{\mathrm{KL}}\!\bigl( \htmlClass{k-policy}{\pi_\theta(\cdot|x)} \;\|\; \htmlClass{k-ref}{\pi_{\mathrm{ref}}(\cdot|x)} \bigr)}\]
The penalty is the KL divergence to the reference model (cyan) — the frozen SFT checkpoint. This measures how far $\htmlClass{k-policy}{\pi_\theta}$ has drifted from its starting point.

Final RLHF objective. $\htmlClass{k-kl}{\beta}$ is the "KL budget" — larger $\htmlClass{k-kl}{\beta}$ → stays close to $\htmlClass{k-ref}{\pi_{\mathrm{ref}}}$; smaller $\htmlClass{k-kl}{\beta}$ → chases $\htmlClass{k-reward}{r_\theta}$.

The KL Leash: β Controls the Trade-off

The reward model $r_\theta$ is a proxy for human preferences. A powerful optimizer exploits its blind spots (reward hacking). The KL term bounds how far $\htmlClass{k-policy}{\pi_\theta}$ can drift, limiting exploitation.

Large β
stays near π_ref
safe, limited gain
⇄
Small β
chases reward
more gain, more risk

β = 0 → pure reward max; β → ∞ → model never moves. The KL budget is a key training hyperparameter.

Recap & What Comes Next

The LLM MDP, in one picture

  • State = prompt + tokens so far
  • Action = next token from $\mathcal{V}$
  • Policy = πθ, the fine-tuned LLM
  • Reward = terminal, scored by learned RM on full $y$
  • KL penalty anchors to π_ref with strength β
  • Objective: $\max_{\htmlClass{k-policy}{\pi_\theta}}\,\mathbb{E}[\htmlClass{k-reward}{r_\theta}] \htmlClass{k-kl}{- \beta D_{\mathrm{KL}}(\htmlClass{k-policy}{\pi_\theta}\|\htmlClass{k-ref}{\pi_{\mathrm{ref}}})}$

Optimization toolkit

Instruction Fine-Tuning (SFT) ↓ Reward Modeling ↓ Policy Gradients / PPO / GRPO ↓ Direct Alignment (DPO)
Every algorithm we study navigates the reward vs. KL tension differently. Keep the objective $\max\,\mathbb{E}[r] - \beta D_{\mathrm{KL}}$ in mind as the shared north star.
