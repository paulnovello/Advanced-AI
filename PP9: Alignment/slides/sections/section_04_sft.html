Instruction Fine-Tuning (SFT)

How a base language model learns to follow instructions — and why that makes every downstream alignment method possible.

Chat Templates: Turning Messages into a Token Sequence

A chat template wraps each message in special tokens and concatenates everything into a single flat sequence the LM trains on autoregressively.

Three canonical roles: system (hidden instructions), user (human turn), assistant (model response). Special tokens <|im_start|> / <|im_end|> delimit every message.

SFT Loss: Derivation Chain

\[ \prod_{t=1}^{T} \htmlClass{k-policy}{\pi_\theta}\!\bigl(\htmlClass{k-adv}{y_t} \mid x, y_{<t}\bigr) \]
Start: the model's likelihood of the full response as a product of per-token conditional probabilities.
\[ \log \prod_{t=1}^{T} \htmlClass{k-policy}{\pi_\theta}\!\bigl(\htmlClass{k-adv}{y_t} \mid x, y_{<t}\bigr) = \sum_{t=1}^{T} \htmlClass{k-logp}{\log \pi_\theta\!\bigl(y_t \mid x, y_{<t}\bigr)} \]
Take log: product becomes a sum — numerically stable and additive.
\[ \htmlClass{k-kl}{\mathcal{L}_{\text{SFT}}} = -\frac{1}{\sum_t \htmlClass{k-kl}{m_t}}\sum_{t=1}^{T} \htmlClass{k-kl}{m_t} \cdot \htmlClass{k-logp}{\log \htmlClass{k-policy}{\pi_\theta}\!\bigl(\htmlClass{k-adv}{y_t} \mid x, y_{<t}\bigr)} \]
Negate (NLL = cross-entropy) and add the loss mask $\htmlClass{k-kl}{m_t}\!\in\!\{0,1\}$: 0 on prompt tokens, 1 on response tokens. Same loss as pretraining — just on a masked subset.

Loss Mask: Where Gradients Flow

In a packed training sequence the prompt tokens are masked out ($m_t = 0$); only response tokens contribute to the gradient.

Full training sequence (prompt + response concatenated):
<|im_start|> system You are a pirate <|im_end|> <|im_start|> user How many helicopters … <|im_end|> <|im_start|> assistant Arrr , I 'd wager no more than half a rotor blade , matey ! <|im_end|>
 token  prompt / special — masked $m_t = 0$  token  response — trained $m_t = 1$

Prompt masking

The model should answer prompts, not memorise them. Masking prompt tokens prevents fitting user-query patterns that don't generalise — only the completion quality matters.

Multi-turn masking

Two conventions: (1) final-turn only — mask all prior turns; (2) mask user turns only — train on every assistant turn. Choice depends on whether intermediate replies are high-quality enough to learn from.

Why SFT Is the Necessary First Step

Teaches Format

A base LM echoes the prompt pattern; an SFT model terminates cleanly at <|im_end|> and follows role structure consistently. Preference data collection and RL optimisation both assume this behaviour.

Sets πref

The SFT model becomes the reference policy πref that all downstream methods — PPO, DPO, GRPO — anchor to via KL divergence. Without a sensible starting point the KL constraint enforces nothing useful.
